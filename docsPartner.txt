═══════════════════════════════════════════════════════════════════
  APA ITU GISTREAM
═══════════════════════════════════════════════════════════════════

GiStream adalah platform streaming JKT48 milik JKT48Connect.
Partner perlu generate JWT token menggunakan HMAC proof-of-knowledge
sebelum bisa memutar stream HLS.

  Token API    : https://v5.jkt48connect.com
  Stream Worker: https://ctv.jkt48connect.com

TIDAK ADA proxy layer. Panggil langsung ke dua domain di atas.

═══════════════════════════════════════════════════════════════════
  CREDENTIAL
═══════════════════════════════════════════════════════════════════

  PARTNER_KID    = "jkt48connect-v1"
                   → non-rahasia, boleh hardcode di kode

  PARTNER_SECRET = [nilai dari tim GiStream]
                   → SANGAT RAHASIA
                   → HANYA boleh ada di: environment variable server,
                     secrets manager (Vault, AWS Secrets Manager,
                     Cloudflare Workers secret, Railway secret)
                   → TIDAK BOLEH ada di: source code, .env yang
                     di-commit ke git, frontend bundle, APK/IPA,
                     variable dengan prefix NEXT_PUBLIC_ / VITE_ /
                     REACT_APP_ / EXPO_PUBLIC_

═══════════════════════════════════════════════════════════════════
  CARA KERJA AUTH — HMAC PROOF-OF-KNOWLEDGE
═══════════════════════════════════════════════════════════════════

Secret TIDAK dikirim. Server GiStream membuktikan kamu tahu
secret-nya dari signature yang valid — mirip AWS Signature V4.

  Formula:
    body_hash   = SHA256("{}")     ← SELALU hash dari string literal "{}"
    signing_str = timestamp + ":" + nonce + ":" + "POST" + ":"
                  + "/api/token/generate?apikey=JKTCONNECT" + ":"
                  + body_hash
    signature   = HMAC-SHA256(signing_str, PARTNER_SECRET) → hex

  Aturan nilai:
    timestamp → Date.now() dalam MILLISECONDS (bukan detik)
    nonce     → crypto.randomUUID() tanpa strip, SEKALI PAKAI,
                tidak boleh dipakai ulang dalam window 60 detik
    body_hash → selalu dari string "{}", bukan object kosong

  Headers yang dikirim ke Token API (POST):
    x-kid        : PARTNER_KID       ← non-rahasia
    x-timestamp  : timestamp ms
    x-nonce      : nonce unik
    x-signature  : HMAC hex          ← BUKAN secret-nya
    x-showid     : show ID           ← atau x-slug jika pakai slug
    Content-Type : application/json
    Body         : {}

  TIDAK ADA header x-partner-secret — header ini sudah dihapus
  di v2.0 dan akan diabaikan server jika dikirim.

═══════════════════════════════════════════════════════════════════
  FLOW LENGKAP
═══════════════════════════════════════════════════════════════════

  1. Generate JWT token:
     POST https://v5.jkt48connect.com/api/token/generate?apikey=JKTCONNECT
     → dapat JWT string

  2. Ambil stream URL:
     GET https://ctv.jkt48connect.com/stream?showId={showId}
     Header: x-api-token: {JWT}
     → dapat array streams[]

  3. Ambil URL stream dari field "url":
     streams[0].url  ← ini yang dipakai untuk HLS player

  4. HLS player load URL tersebut.
     Field "url" TETAP membutuhkan header x-api-token di setiap
     HLS request (manifest dan segment) — sama seperti "url-2".
     Inject via xhrSetup dan fetchSetup di konfigurasi hls.js.

  PENTING — Token bersifat ONE-TIME-USE:
  Setelah endpoint /stream dipanggil, JTI token di-blacklist
  di KV storage. JANGAN panggil /stream dua kali dengan token
  yang sama. Generate token baru untuk setiap sesi baru.

Token berlaku 2 jam (7200 detik). Untuk livestream theater yang
bisa berlangsung >2 jam, implementasikan refresh:

═══════════════════════════════════════════════════════════════════
  🔐 SECURITY — WAJIB DIIKUTI (JANGAN SKIP)
═══════════════════════════════════════════════════════════════════

Ini yang PALING PENTING. Pastikan semua poin ini ada di kode:

  CRITICAL — jika dilanggar, secret bisa bocor:

  ✦ PARTNER_SECRET hanya dari environment variable di sisi server.
    Jangan pernah taruh nilainya langsung di kode.
    Benar : process.env.PARTNER_SECRET
    Salah : const SECRET = "gstream@..."  ← BAHAYA, akan bocor

  ✦ Jangan pernah expose PARTNER_SECRET ke response API manapun.
    Jangan log, jangan print, jangan masukkan ke error message.

  ✦ Jika stack kamu adalah React/Next.js, pastikan HMAC signing
    dilakukan HANYA di server-side (API Route, Server Action,
    getServerSideProps). JANGAN di client component.

  ✦ Jika stack adalah Cloudflare Workers / Hono, simpan secret
    via: wrangler secret put PARTNER_SECRET
    JANGAN masukkan ke wrangler.toml atau file yang di-commit.

  ✦ Jika stack adalah Vite/CRA/Lovable (pure frontend), WAJIB
    buat backend security alias terlebih dahulu (lihat Prompt B).
    PARTNER_SECRET tidak boleh ada di file yang di-bundle ke browser.

  ✦ JWT token GiStream mengandung x-api-token yang dipakai untuk
    mengakses HLS stream. Jaga token ini sama ketatnya dengan
    menjaga PARTNER_SECRET — token yang bocor bisa dipakai orang
    lain untuk menonton stream hingga expired (2 jam).

  IMPORTANT — mitigasi risiko tambahan:

  ✦ Tambahkan autentikasi user sebelum memanggil generate token.
    Endpoint yang memanggil GiStream Token API harus dilindungi —
    hanya user yang punya akses (tiket/membership) yang boleh
    mendapatkan token.

  ✦ Tambahkan rate limiting pada endpoint generate token.
    Batasi misalnya 5 request/menit per user_id atau per IP.
    Tanpa ini, kuota token partner bisa dihabiskan oleh abuse.

  ✦ Jangan simpan JWT GiStream di localStorage atau sessionStorage.
    Simpan di React state (memory) atau httpOnly cookie.
    JWT di localStorage rentan dicuri via XSS.

  ✦ Jangan log JWT token ke console atau server log di production.
    Token bisa di-replay sampai expired jika bocor dari log.

  ✦ Jika menggunakan mobile app (React Native/Flutter):
    - PARTNER_SECRET tidak boleh ada di APK/IPA
    - APK bisa di-decompile dengan jadx/apktool
    - Signing harus dilakukan di backend, app hanya terima JWT jadi
    - Gunakan certificate pinning untuk channel app <-> backend

  ✦ Lakukan rotasi PARTNER_SECRET secara berkala (setiap 90 hari)
    atau segera jika ada indikasi kebocoran. Koordinasikan dengan
    tim GiStream untuk mendapatkan kid baru via zero-downtime
    rotation (kid lama dan baru aktif bersamaan selama transisi).

  ✦ Scan repository dengan gitleaks atau truffleHog sebelum push
    untuk mendeteksi secret yang tidak sengaja ter-commit.

  ✦ Gunakan .gitignore yang benar — pastikan .env, .dev.vars,
    dan file secrets lokal tidak pernah masuk ke repository.

═══════════════════════════════════════════════════════════════════
  ERROR CODES YANG MUNGKIN MUNCUL
═══════════════════════════════════════════════════════════════════

  Generate Token (v5.jkt48connect.com):
    401 → Header tidak lengkap / timestamp expired (>60 detik) /
          HMAC signature tidak cocok / x-kid tidak ditemukan
          (semua error ini disatukan pesannya untuk mencegah
           attacker enumerate kid yang valid)
    403 → Nonce sudah pernah dipakai (replay attack detected)
    400 → Tidak ada x-slug atau x-showid / apikey tidak disertakan
    429 → Rate limit exceeded (30 req/menit per IP)

  Stream Worker (ctv.jkt48connect.com):
    401 TOKEN_EXPIRED     → Generate token baru
    403 INVALID_SIGNATURE → Token dimanipulasi
    403 JTI_USED          → Token sudah dikonsumsi sebelumnya
    403 IP_MISMATCH       → IP berubah setelah token dibuat
    403 SLUG/SHOWID_MISMATCH → Identifier tidak cocok dengan token
    403 INVALID_SECRET    → Partner secret hash tidak cocok
    502                   → Stream tidak ditemukan / CDN error

  Debugging 401 signature:
    1. Pastikan timestamp dalam MILLISECONDS (Date.now(), bukan /1000)
    2. Pastikan body yang di-hash adalah string "{}" (bukan object)
    3. Pastikan signing path menyertakan ?apikey=JKTCONNECT
    4. Pastikan PARTNER_SECRET sama persis (tidak ada trailing space)

--- Helpers (jalankan di sisi server/backend SAJA) ---

const PARTNER_KID    = "jkt48connect-v1";           // non-rahasia
const PARTNER_SECRET = process.env.PARTNER_SECRET!; // HANYA env var
const TOKEN_API      = "https://v5.jkt48connect.com";
const CTV_BASE       = "https://ctv.jkt48connect.com";
const SIGNING_PATH   = "/api/token/generate?apikey=JKTCONNECT";

async function sha256Hex(str: string): Promise<string> {
  const buf = await crypto.subtle.digest(
    "SHA-256", new TextEncoder().encode(str)
  );
  return Array.from(new Uint8Array(buf))
    .map(b => b.toString(16).padStart(2, "0")).join("");
}

async function hmacSHA256Hex(secret: string, msg: string): Promise<string> {
  const key = await crypto.subtle.importKey(
    "raw", new TextEncoder().encode(secret),
    { name: "HMAC", hash: "SHA-256" }, false, ["sign"]
  );
  const buf = await crypto.subtle.sign(
    "HMAC", key, new TextEncoder().encode(msg)
  );
  return Array.from(new Uint8Array(buf))
    .map(b => b.toString(16).padStart(2, "0")).join("");
}

async function buildHMACHeaders(): Promise<Record<string, string>> {
  const timestamp  = Date.now().toString();  // MILLISECONDS
  const nonce      = crypto.randomUUID().replace(/-/g, "");
  const bodyHash   = await sha256Hex("{}");  // hash dari string "{}"
  const signingStr = `${timestamp}:${nonce}:POST:${SIGNING_PATH}:${bodyHash}`;
  const signature  = await hmacSHA256Hex(PARTNER_SECRET, signingStr);
  return {
    "x-kid":       PARTNER_KID,  // non-rahasia
    "x-timestamp": timestamp,
    "x-nonce":     nonce,
    "x-signature": signature,
    // TIDAK ADA "x-partner-secret" — dihapus di v2.0
  };
}

--- Generate Token ---

async function generateGiStreamToken(
  showId: string,
  isSlug: boolean
): Promise<string> {
  const hmacHeaders = await buildHMACHeaders();
  const res = await fetch(`${TOKEN_API}${SIGNING_PATH}`, {
    method: "POST",
    headers: {
      ...hmacHeaders,
      ...(isSlug ? { "x-slug": showId } : { "x-showid": showId }),
      "Content-Type": "application/json",
    },
    body: "{}",
  });
  const text = await res.text();
  let data: any;
  try { data = JSON.parse(text); }
  catch { throw new Error("Token API non-JSON response"); }
  if (!data.status) throw new Error("Generate token gagal: " + data.message);
  return data.data.token as string;
}

--- Get Stream URL (gunakan field "url") ---

interface GiStreamQuality {
  index:     number;
  name:      string;
  bandwidth: number;
  url:       string;   // ← field "url", ini yang dipakai untuk player
}

async function getGiStreamURL(
  token: string,
  showId: string,
  isSlug: boolean
): Promise<{ url: string; qualities: GiStreamQuality[] }> {
  const param = isSlug ? `slug=${showId}` : `showId=${showId}`;
  const res = await fetch(`${CTV_BASE}/stream?${param}`, {
    headers: {
      "x-api-token": token,
      ...(isSlug ? { "x-slug": showId } : { "x-showid": showId }),
    },
  });
  const data = await res.json();
  if (!data.success) throw new Error(data.message || "Gagal ambil stream URL");

  const streams: any[] = data.streams || [];
  const qualities: GiStreamQuality[] = streams.map((s, idx) => ({
    index:     idx,
    name:      s.NAME || `q${idx}`,
    bandwidth: parseInt(s.BANDWIDTH) || 0,
    url:       s.url || "",   // ← field "url"
  }));

  return {
    url: streams[0]?.url || "",  // ← field "url" dari stream[0]
    qualities,
  };
}

--- Token Refresh Manager ---

class GiStreamTokenManager {
  private token: string | null = null;
  private timer: ReturnType<typeof setInterval> | null = null;

  constructor(
    private readonly showId: string,
    private readonly isSlug: boolean,
    private readonly onRefresh: (url: string, token: string) => void
    // callback untuk re-init hls.js dengan token + url baru
  ) {}

  async start(): Promise<{ url: string; token: string }> {
    await this._refresh();
    this.timer = setInterval(() => this._checkExpiry(), 5 * 60 * 1000);
    const { url } = await getGiStreamURL(this.token!, this.showId, this.isSlug);
    return { url, token: this.token! };
  }

  private async _refresh() {
    this.token = await generateGiStreamToken(this.showId, this.isSlug);
    // Ambil URL baru dengan token baru, karena URL terikat token
    const { url } = await getGiStreamURL(this.token, this.showId, this.isSlug);
    // Re-init hls.js dengan token baru agar xhrSetup pakai token terbaru
    this.onRefresh(url, this.token);
  }

  private async _checkExpiry() {
    if (!this.token) return;
    try {
      const payload = JSON.parse(
        atob(this.token.split(".")[1].replace(/-/g, "+").replace(/_/g, "/"))
      );
      const timeLeft = payload.exp - Math.floor(Date.now() / 1000);
      if (timeLeft <= 300) await this._refresh();
    } catch { /* ignore */ }
  }

  stop() {
    if (this.timer) clearInterval(this.timer);
    this.token = null;
  }
}

⚠️ BACA INI DULU SEBELUM PAKAI PROMPT B:

Lovable adalah tools yang generate React frontend. Karena semua
kode React di-bundle ke browser dan bisa dilihat siapa saja,
PARTNER_SECRET tidak boleh ada di kode yang Lovable generate.

Solusi: Lovable akan membuat DUA bagian:
  1. Backend security alias (Node.js/Express kecil) yang menyimpan
     PARTNER_SECRET dan melakukan HMAC signing. Deploy ini terpisah
     di Railway, Render, atau platform sejenis.
  2. React frontend yang hanya tahu URL backend alias tersebut —
     tidak tahu PARTNER_SECRET sama sekali.

Arsitektur yang benar:
  Browser (React) → Backend Alias (Node.js) → GiStream API
                  token + url dikembalikan ke React
                  React inject token ke HLS requests

    "expiresAt": "2025-05-15T10:00:00.000Z"
  }

  Response error:
  { "success": false, "error": "pesan error yang aman" }
    ← JANGAN expose detail internal atau nilai PARTNER_SECRET
      di error message

Cara backend memanggil GiStream:

  Step 1 — Generate JWT token:
    POST https://v5.jkt48connect.com/api/token/generate?apikey=JKTCONNECT
    Headers:
      x-kid        : "jkt48connect-v1"
      x-timestamp  : Date.now().toString()    ← MILLISECONDS
      x-nonce      : crypto.randomUUID() tanpa strip
      x-signature  : HMAC-SHA256 hex (lihat formula di bawah)
      x-showid     : showId  (atau x-slug jika isSlug=true)
      Content-Type : application/json
    Body: {}

  Formula HMAC signature:
    body_hash   = SHA256("{}")
    signing_str = timestamp + ":" + nonce + ":" + "POST" + ":"
                  + "/api/token/generate?apikey=JKTCONNECT" + ":"
                  + body_hash
    signature   = HMAC-SHA256(signing_str, process.env.PARTNER_SECRET)
                  → return sebagai hex string

  Step 2 — Ambil stream URL:
    GET https://ctv.jkt48connect.com/stream?showId={showId}
    Header: x-api-token: {JWT dari step 1}
    → dari response, ambil streams[0].url (field "url")
    → return token + url + qualities ke frontend

  PENTING: backend TIDAK melakukan proxy stream.
  Backend hanya return token dan URL ke frontend.
  Frontend yang akan langsung hit URL tersebut dengan
  menginject x-api-token ke setiap HLS request.

Konfigurasi keamanan backend yang WAJIB:

  - CORS: hanya izinkan origin dari domain frontend Lovable kamu
    Contoh: origin: ["https://your-app.lovable.app"]
    JANGAN pakai origin: "*" — ini membuka akses dari mana saja

  - Rate limiting: maksimal 10 request/menit per IP
    Gunakan express-rate-limit

  - Helmet.js: tambahkan security headers standar

  - Environment variables yang perlu di-set di Railway/Render:
    PARTNER_SECRET = [nilai dari tim GiStream]   ← RAHASIA
    PARTNER_KID    = jkt48connect-v1             ← non-rahasia
    ALLOWED_ORIGIN = https://your-app.lovable.app

  - JANGAN commit .env ke git. Tambahkan ke .gitignore.

  - JANGAN pernah log atau print PARTNER_SECRET, bahkan di dev mode.

  - Validasi input showId sebelum dipakai — jangan langsung
    forward nilai mentah dari request ke GiStream API.

  - JANGAN log nilai token JWT di server log production.

  Dependencies yang dibutuhkan (package.json):
    express, cors, helmet, express-rate-limit, dotenv
